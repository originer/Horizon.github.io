记录一个redis集群相关的问题

发表于 2019-11-12 | 分类于 redis

问题描述

一个由六个节点组成的redis集群，部署在三台机器上，每台机器分别放一主一从，但是由于各种原因，会导致发生主从切换，此时集群状态不受影响，但是出现了同一台机器上有两个master的情况。

为什么要这样部署：节省服务器成本

一台机器的节点是双master的风险

master节点负责读写，双master都在一台机器工作会提高机器的cpu压力。

如果出现机器宕机，由于双主同时下线，就没有时间进行重新投票产生新的master，集群就会直接变成不可用状态，此时，集群就不是高可用状态。

解决方案

通过脚本，定时轮询，调整主从节点位置，避免双主在一台机器上。
