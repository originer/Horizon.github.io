依赖查找与依赖注入的区别？

依赖查找和依赖注入都是IOC实现的方式，区别在于，依赖查找是通过框架提供的API进行手动查找，依赖注入是通过手动或者自动的方式把配置好的Bean与声明进行绑定，依赖查找侧重查找，依赖注入侧重于绑定。一般工作中，依赖注入使用的比较多。

pwa实践总结

发表于 2020-09-23 | 分类于 前端

pwa是谷歌提出一套规范，当网站实现以下要求时，就可以被称为是pwa网站。

  • 站点需要使用 HTTPS
  • 页面需要响应式，能够在平板和移动设备上都具有良好的浏览体验
  • 所有的 URL 在断网的情况下有内容展现，不会展现浏览器默认页面
  • 需要支持 Wep App Manifest，能被添加到桌面
  • 即使在 3G 网络下，页面加载要快，可交互时间要短
  • 在主流浏览器下都能正常展现
  • 动画要流畅，有用户操作反馈
  • 每个页面都有独立的 URL

pwa的核心特性

可安装

可安装特性在不同的浏览器上的效果是不同的，比如chrome是有一个弹窗，三星只有一个角标，ios上则没有任何提示，只能用户通过浏览器扩展按钮自行安装。

要求：

1、manifest.json文件配置完整；

2、serviceWorker可以激活并运行；

可离线工作 serviceWorker

利用serviceworker的缓存，让网站在离线时也有部分功能可用。

作用：

类似一个代理服务器，可以拦截所有的http请求。我们可以选择缓存请求，这样在下次进行访问的时候会优先从serviceWorker缓存中取数据。

redis缓存淘汰机制总结

发表于 2020-03-11 | 分类于 redis

##

1、有设置expire过期时间的key

主动删除

redis会定时轮询，删除过期的key，轮询间隔可以通过hz配置；

惰性删除

过期的key一般还会持久化在内存中，当被再次访问后redis会检测是否过期，如果已经过期就会进行删除；

2、没设置expire的key 内存淘汰管理机制

redis在内存使用超过设置的阈值后会自动清理

具体配置：memory-policy 可以根据具体业务进行配置

volatile.lru .> Evict using approximated LRU among the keys with an expire set.
allkeys.lru .> Evict any key using approximated LRU.
volatile.lfu .> Evict using approximated LFU among the keys with an expire set.
allkeys.lfu .> Evict any key using approximated LFU.
volatile.random .> Remove a random key among the ones with an expire set.
allkeys.random .> Remove a random key, any key.
volatile.ttl .> Remove the key with the nearest expire time (minor TTL)
noeviction .> Don’t evict anything, just return an error on write operations.

JMM内存模型学习笔记

发表于 2019-08-08 | 分类于 Java

java内存模型

注意：此处指的内存模型是java与操作系统交互的内存模型，不是java的虚拟机内存模型（jvm），jvm是java虚拟机内存模型的一种规范

线程之间无法传递数据，一定要经过主内存，所以要保持同步最简单的做法是对主内存的数据上锁。但是就变成了单线程十分影响效率，所以后来添加了总线嗅探机制，当别的线程嗅探到数据改变了，会把工作内存中的数据置为失效，然后重新从主内存中读取数据。

cpu早期保持数据一致性的做法

总线

总线嗅探机制

mesi缓存一致性协议

volatile

带有volatile修饰的变量的底层实现是用汇编指令lock前缀修饰，lock有两个作用：

1、当变量值改变时立即写回主内存，同时对主内存上锁；

2、同时把在总线中的相同变量置为无效状态（MESI协议）

volatile无法保证原子性

区别

volatile的加锁时机与总线的加锁不一样，总线是在read的时候就需要上锁，volatile只有在修改值的时候才上锁。
